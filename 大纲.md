# gloal_verifier 大纲
- 本项目分成前后端
    - 前端包含一个agent，用来同环境交互
        - 前端在碰到一个新的state的时候，会去问后端的experience base有没有相关的experience，experience base在判断在之后会去给experience
        - 前端在state s_t的时候，输入 action a, 状态转换为下一个 state s_{t+1}；将这一串输入给experience base，让experience base把它们组成新的experience
    - 后端包含一个experience base，
        - 后端可能包含一个agent，用来在前端需要的时候给前端反应experience，总结experience，判断experience是否contradict 等等

## 后端 experience base
- 收到前端的请求回去找合适的experience输出
- 收到前端的信息，会整理experience存入
    - 和现有信息有冲突，会去记录下来，等前端空闲，做n次实验验证
- experience的格式是
    - (s_t, a, s_{t+1})
        - s_t 初始状态
        - a action
        - s_{t+1} 从 s_t 出发，执行过 a 之后的下一个状态
- 可能会添加 llm agent
